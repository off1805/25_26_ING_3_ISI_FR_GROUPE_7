DataTable - Exemple Backend

Démonstration des appels backend automatiques pour recherche, filtres, tri et pagination

Exemple 1: Utilisateurs (Backend)

Chaque changement (recherche, filtre, tri, pagination) déclenche un appel API au backend

Code JavaScript

// Configuration du DataTable avec backend const table = new DataTable({ containerId: 'users-table-backend', title: 'Gestion des utilisateurs', isRemoteData: true, // ← Mode backend activé columns: [ { key: 'email', label: 'Utilisateur', type: 'avatar', sortable: true, getter: (item) => item.email, }, { key: 'roleName', label: 'Rôle', type: 'badge', sortable: true, filterable: true, filterType: 'select', filterOptions: [ { value: 'TEACHER', label: 'Enseignant' }, { value: 'SURVEILLANT', label: 'Surveillant' }, { value: 'AP', label: 'Assistant pédagogique' }, ], }, // ... autres colonnes ], // Callback de recherche onSearch: async (query, filters, sort, page, itemsPerPage) => { const response = await fetch('/api/users/search', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ query, filters, sort, page, limit: itemsPerPage, }), }); return await response.json(); // Format attendu: { data: [...], totalItems: 123 } }, // Callback de filtre onFilter: async (filters, search, sort, page, itemsPerPage) => { const response = await fetch('/api/users/filter', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ filters, search, sort, page, limit: itemsPerPage, }), }); return await response.json(); }, // Callback de tri onSort: async (sort, search, filters, page, itemsPerPage) => { const response = await fetch('/api/users/sort', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ sort: { key: sort.key, direction: sort.direction }, search, filters, page, limit: itemsPerPage, }), }); return await response.json(); }, // Callback de changement de page onPageChange: async (page, itemsPerPage, search, filters, sort) => { const response = await fetch('/api/users/page', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ page, limit: itemsPerPage, search, filters, sort, }), }); return await response.json(); }, // Callback de changement du nombre d'items par page onItemsPerPageChange: async (itemsPerPage, page, search, filters, sort) => { const response = await fetch('/api/users/items-per-page', { method: 'POST', headers: { 'Content-Type': 'application/json' }, body: JSON.stringify({ limit: itemsPerPage, page, search, filters, sort, }), }); return await response.json(); }, // Actions locales (pas d'appel backend ici) onAction: (actionType, user) => { if (actionType === 'edit') { console.log('Éditer:', user); } else if (actionType === 'delete') { console.log('Supprimer:', user); } }, });

Endpoints Backend Attendus

POST /api/users/search

Recherche globale dans les utilisateurs

Request: { "query": "alice", "filters": { "role": "TEACHER", "status": "ACTIVE" }, "sort": { "key": "email", "direction": "asc" }, "page": 1, "limit": 50 } Response: { "data": [ { "id": 1, "email": "alice@example.com", "roleName": "TEACHER", ... }, ... ], "totalItems": 123 }

POST /api/users/filter

Filtre avec les paramètres actuels

Request: { "filters": { "role": "TEACHER" }, "search": "alice", "sort": null, "page": 1, "limit": 50 } Response: { "data": [...], "totalItems": 45 }

POST /api/users/sort

Tri avec les paramètres actuels

Request: { "sort": { "key": "email", "direction": "desc" }, "search": "", "filters": {}, "page": 1, "limit": 50 } Response: { "data": [...], "totalItems": 150 }

POST /api/users/page

Changement de page avec les paramètres actuels

Request: { "page": 2, "limit": 50, "search": "alice", "filters": { "role": "TEACHER" }, "sort": { "key": "email", "direction": "asc" } } Response: { "data": [...], "totalItems": 45 }

✨ Avantages

  • 🚀 Performance: Chargement à la demande, gestion serveur de grandes collections
  • 🔍 Recherche serveur: Recherche sur la base de données complète, pas juste les données visibles
  • 📊 Tri serveur: Tri efficace sur le backend, optimisé par les index DB
  • ⚡ Pagination: Ne charge que les données nécessaires pour la page actuelle
  • 🔄 Cohérence: Les données sont toujours fraîches, directement du serveur
  • 🛠️ Flexible: Adaptez la logique backend selon vos besoins métier

📋 Cas d'Usage

Données volumineuses

Millions d'enregistrements: isRemoteData: true pour ne charger que ce qui est visible

Données en temps réel

Les données changent souvent: le backend fournit toujours les données fraîches

Permissions complexes

Les données filtrées dépendent des permissions de l'utilisateur: le serveur gère la logique

Calculs complexes

Les résultats nécessitent des calculs serveur: onSort/onFilter exécute la logique métier
